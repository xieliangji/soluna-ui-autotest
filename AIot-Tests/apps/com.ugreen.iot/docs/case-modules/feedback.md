# 意见反馈用例

本模块记录“我的-意见反馈”相关用例。

## 模块规则

- 历史反馈入口和反馈页返回图标是 WebView 视觉入口，Android/iOS 都保持使用 `tapVisualTemplate`。
- 历史图标优先使用标题文本元素的 `y` 和 `height` 派生整屏宽度标题栏 ROI，不要写成很窄的固定右上角 ROI；返回图标当前使用页面 header 左上角 ROI。
- 历史图标模板必须紧裁到图标本体，避免模板高度大于标题栏 ROI 或白底区域过大导致误匹配标题文字。
- 历史列表和详情页必须等待第一条历史记录出现，不要只断言通用 WebView/list 容器。
- Android 和 iOS 描述输入超长行为不同：iOS 截断，Android 显示 `201/200` 并允许提交后 toast 失败。
- toast 断言优先收窄 ROI；OCR 后尽量增加结构性断言，例如仍停留在意见反馈页。
- 设备选择依赖当前账号存在可选设备；没有设备时放入 focused debug plan，不放入常规聚合计划。

## 用例记录

### TC009 我的-意见反馈-访客模式访问

状态：已实现，游客态用例。

操作路径：

1. 点击“我的”tab。
2. 断言“我的”页列表可见。
3. 断言意见反馈入口可见。
4. 点击意见反馈入口。
5. 断言意见反馈页可见。

验证点：

- 游客态可以看到意见反馈入口。
- 游客态可以进入意见反馈页。

### TC010 我的-意见反馈-提交成功

状态：已实现，已登录态用例。

操作路径：

1. 点击“我的”tab。
2. 点击意见反馈入口。
3. 断言意见反馈页可见。
4. 选择问题类型。
5. 输入有效反馈内容。
6. 点击问题类型区域收起描述输入焦点。
7. 开始录屏。
8. 点击提交反馈。
9. 等待 toast 展示窗口。
10. 停止录屏并保存录屏资源。
11. 对录屏帧做 OCR，校验提交成功提示。
12. 断言提交成功后返回“我的”页。

验证点：

- 有效反馈内容可提交成功。
- 提交成功 toast 可识别。
- 提交成功后回到“我的”页。

注意事项：

- toast 显示时间较短，使用录屏帧 OCR 而不是单张截图。
- iOS 全量中 toast 偶发受提交响应和录屏首帧时序影响，录屏窗口保持 5 秒，帧上限同步放大，并放宽 toast ROI，避免漏掉成功提示。

### TC011 我的-意见反馈-问题类型

状态：已实现。

### TC012 我的-意见反馈-设备

状态：已实现，但依赖当前账号存在可选设备；当前常规计划可按设备前置条件决定是否纳入。

前置条件：

- 已登录账号存在至少一个可选设备。
- 选择设备相关问题类型后，意见反馈页展示设备选择栏。

操作路径：

1. 点击“我的”tab。
2. 点击意见反馈入口。
3. 断言意见反馈页可见。
4. 选择设备相关问题类型。
5. 断言设备选择栏可见。
6. 第一次进入设备选择页。
7. 断言设备列表可见。
8. 断言至少存在一个可选设备。
9. 使用返回图标模板返回意见反馈页。
10. 断言已回到意见反馈页。
11. 再次进入设备选择页。
12. 记录第一个设备名称。
13. 选择第一个设备。
14. 点击完成/确认设备选择。
15. 断言设备栏显示已选设备名称。

验证点：

- 设备选择页可进入。
- 可选择设备并回填到反馈页设备栏。

注意事项：

- 没有可选设备时不要将该用例放入常规全量计划。

### TC013 我的-意见反馈-问题描述或建议

状态：已实现，iOS/common 行为。

操作路径：

1. 点击“我的”tab。
2. 点击意见反馈入口。
3. 断言意见反馈页可见。
4. 选择问题类型。
5. 聚焦问题描述输入框。
6. 输入少于 10 个字符的问题描述。
7. 收起输入焦点。
8. 断言提交按钮不可用。
9. 输入 10 个字符的问题描述。
10. 收起输入焦点并提交。
11. 断言提交成功后返回“我的”页。
12. 再次进入意见反馈页。
13. 选择问题类型。
14. 输入 200 个字符的问题描述。
15. 断言计数显示 `200/200`。
16. 收起输入焦点并提交。
17. 断言提交成功后返回“我的”页。
18. 第三次进入意见反馈页。
19. 选择问题类型。
20. 输入 201 个字符的问题描述。
21. 断言超过 200 字符后计数仍保持上限。

验证点：

- 少于 10 字符不可提交。
- 10 字符可提交。
- 200 字符可提交。
- iOS/common 超长输入按上限截断或保持上限计数。

### TC013 我的-意见反馈-问题描述或建议-Android

状态：已实现，Android 专用行为。

操作路径：

1. 执行 TC013 common 前半段：少于 10 字符不可提交、10 字符提交成功、200 字符提交成功。
2. 第三次进入意见反馈页。
3. 选择问题类型。
4. 输入 201 个字符的问题描述。
5. 断言 Android 显示超限计数 `201/200`。
6. 断言 Android 超限时提交按钮仍可用。
7. 开始录制超限提交失败提示。
8. 点击提交 201 字符反馈。
9. 等待失败 toast 展示窗口。
10. 停止录屏并保存录屏资源。
11. 使用多模态 OCR 校验 Android 超限提交失败提示。
12. 断言失败后仍停留在意见反馈页。

验证点：

- Android 超长输入显示 `201/200`。
- Android 超长输入可以点击提交，但提交失败。
- 失败 toast 可识别。
- 失败后页面不离开意见反馈页。

注意事项：

- 该用例使用多模态 OCR，API 配置只放在运行环境中。
- ROI 应收窄到 toast 核心区域，避免 OCR 被页面常驻文本干扰。

### TC014 我的-意见反馈-历史反馈-有反馈记录

状态：已实现。

前置条件：

- 当前账号存在至少一条历史反馈记录。

操作路径：

1. 点击“我的”tab。
2. 点击意见反馈入口。
3. 断言意见反馈页可见。
4. 保存意见反馈标题文本元素的整屏宽度标题栏 ROI。
5. 使用历史图标模板和标题栏 ROI 点击进入历史反馈页。
6. 等待至少一条历史反馈记录出现。
7. 使用返回图标模板返回意见反馈页。
8. 断言已回到意见反馈页。

验证点：

- 历史反馈页可进入。
- 至少存在一条历史反馈记录。
- 可从历史页返回意见反馈页。

注意事项：

- 不要改成通用 button locator。iOS WebView 不暴露历史图标为稳定可访问元素。
- 标题栏 ROI 只使用标题文本元素的 `y` 和 `height`，左右边界扩展为整屏宽度，确保左右图标不在标题元素内部时仍能覆盖。

### TC015 我的-意见反馈-历史反馈-反馈详情

状态：iOS 真机已通过，使用 Android/iOS 共用 case 流程。

前置条件：

- 当前账号存在至少一条历史反馈记录。

操作路径：

1. Stage setup 通过 `appState.loggedInDevicePage` 准备已登录设备页。
2. Case setup 通过 `appState.restartApp` 重启 App。
3. 点击“我的”tab。
4. 点击“意见反馈”入口。
5. 断言意见反馈页面 root 可见。
6. 保存意见反馈标题文本元素的整屏宽度标题栏 ROI。
7. 使用 `tapVisualTemplate` 点击反馈历史图标，模板为历史图标模板，ROI 为标题栏 ROI。
8. 等待第一条历史反馈记录 enabled。
9. 点击第一条历史反馈记录。
10. 断言反馈详情内容可见。
11. 断言反馈详情回复区域可见。
12. 使用 `tapVisualTemplate` 点击反馈返回图标，模板为返回图标模板，ROI 为 header 左上角。
13. 等待第一条历史反馈记录再次 enabled，证明已返回历史列表页。

验证点：

- 历史反馈详情页可进入。
- 详情内容可见。
- 回复区域可见。
- 可返回历史反馈列表。

注意事项：

- 不要把历史/返回模板点击替换成 `feedbackHistoryButton` 或通用 button locator。iOS WebView source 不暴露历史图标为可访问 button，宽泛 button 定位容易点到提交按钮。
- 这里 Android 和 iOS 应保持共用逻辑；模板点击是该 WebView 图标入口的稳定跨平台抽象。
- 标题栏 ROI 只使用标题文本元素的 `y` 和 `height`，左右边界扩展为整屏宽度，确保历史图标不在标题元素内部时仍能覆盖。
- 本路径的 focused debug plan 是 `plans/debug/ios-feedback-tc015-debug.yaml`。
