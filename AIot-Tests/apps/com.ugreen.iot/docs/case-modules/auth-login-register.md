# 登录注册与首次使用用例

本模块记录登录、注册、忘记密码、访客限制和首次使用协议相关用例。

## 模块规则

- 验证码相关用例只触发验证码发送，并断言发送按钮进入秒级倒计时；不输入验证码，不重置密码，不完成注册。
- `auth-login-page` stage 使用 `appState.loginPage` 收敛登录页，并在每条 case 前再次回到登录页，避免上一条验证码倒计时或子页面状态影响下一条。
- 密码登录成功用例会改变登录态，应放在登录页 stage 的最后。
- 首次使用协议用例会执行 `clearAppData`，暂不放入常规全量或聚合调试计划。清理应用数据后会丢失登录态、协议同意态和本地授权状态。
- Android `clearAppData` 依赖框架在 `autoGrantPermissions` session 下重新授予 runtime permissions，避免清数据后的首启流程被系统权限弹框打断。
- 首次使用弹框中的“用户协议”和“隐私政策”不是独立稳定元素，当前 Android 用例使用 `common.authDialogContent` 的元素内相对点击比例打开链接。
- 邮箱注册需要切换到“海外测试”地区；用例 teardown 必须恢复到“国内开发”地区，避免影响后续手机号注册或登录页用例。
- 当前 Android 游客页未暴露“账户与安全”入口，访客不支持功能只覆盖头像入口和数据管理入口。
- iOS 密码输入后会额外出现清除按钮，密码明暗文图标定位使用最后一个 eye 图标。
- iOS 验证码发送后倒计时以静态文本展示，发送验证码类用例断言对应 `Ns` 文本节点，不断言原发送按钮。
- iOS 注册页输入密码后需要先点击页面空白区域收起键盘，再勾选协议，避免键盘遮挡协议勾选区域。
- iOS 国家/地区选择列表优先定位可见 table cell；邮箱注册切海外测试后必须恢复国内测试地区。

## 用例记录

### TC024 登录-账号密码登录-密码明暗文功能校验

状态：已实现，iOS/Android 用例。

前置条件：

- Stage 已收敛到登录页。

操作路径：

1. 切换到密码登录。
2. 输入密码。
3. 断言密码输入框默认处于隐藏状态。
4. 点击密码明暗文图标。
5. 断言密码输入框切换为明文状态。
6. 再次点击密码明暗文图标。
7. 断言密码输入框切回隐藏状态。

验证点：

- 密码默认隐藏。
- 明暗文图标可反复切换输入框 `password` 属性。

注意事项：

- iOS 输入密码后会出现清除按钮，明暗文图标使用最后一个 eye 图标定位。

### TC028 注册-未勾选用户协议勾选框

状态：已实现，iOS/Android 用例。

前置条件：

- Stage 已收敛到登录页。

操作路径：

1. 点击注册入口进入注册页。
2. 输入注册账号。
3. 输入注册密码。
4. 未勾选协议时点击发送验证码。
5. 断言协议确认弹框可见。
6. 断言弹框内容包含用户协议、隐私政策和个人信息收集清单。
7. 断言默认展示“同意并继续”按钮。
8. 点击取消关闭弹框。
9. 断言注册账号输入未清空。
10. 断言协议仍未勾选。
11. 再次未勾选协议点击发送验证码。
12. 断言协议确认弹框再次可见。
13. 点击同意并继续。
14. 断言注册验证码按钮进入秒级倒计时。

验证点：

- 未勾选协议会弹出协议确认弹框。
- 取消后注册页数据不清空，协议仍未勾选。
- 同意并继续后可触发验证码发送。

注意事项：

- 当前自动化只验证到验证码倒计时，不完成注册。
- iOS 注册页输入密码后需要先收起键盘，再执行协议相关点击。

### TC030 注册-邮箱注册-发送验证码

状态：已实现，iOS/Android 用例。

前置条件：

- Stage 已收敛到登录页。
- case setup 进入注册页并切换到“海外测试”地区。
- case teardown 恢复到“国内开发”地区。

操作路径：

1. 进入注册页。
2. 如果当前地区不是“海外测试”，打开国家/地区选择弹框。
3. 选择“海外测试”地区并确认；如出现国家/地区切换提示，再次确认。
4. 断言注册页地区已切换为“海外测试”。
5. 输入未注册邮箱。
6. 输入注册密码。
7. 勾选注册协议。
8. 点击发送验证码。
9. 断言发送验证码按钮进入秒级倒计时。
10. teardown 中将注册地区恢复为“国内开发”。

验证点：

- 邮箱注册需要的海外测试地区可切换成功。
- 邮箱注册可触发验证码发送。
- 用例结束后地区恢复。

注意事项：

- 不输入验证码，不完成注册。
- iOS 国家/地区列表使用可见 table cell 定位；切到海外测试并发送验证码后，需要恢复国内测试地区。

### TC032 其他-首次使用用户协议及隐私政策查看

状态：已实现，Android 用例。

前置条件：

- 用例暂不纳入常规全量计划；需要专项验证时使用 `plans/debug/android-first-use-agreement-debug.yaml` 单独执行。
- Android session 建议启用 `autoGrantPermissions`，配合 `clearAppData` 后自动恢复 runtime permissions。

操作路径：

1. 清除应用数据并启动应用。
2. 等待首次启动弹框稳定。
3. 断言首次使用协议弹框标题为个人信息保护条款。
4. 断言弹框内容包含用户协议、隐私政策和同意并继续。
5. 点击取消。
6. 等待应用退出到桌面。
7. 再次启动应用。
8. 断言首次使用协议弹框再次出现。
9. 在弹框内容区域按元素内比例点击用户协议链接。
10. 断言用户协议页面标题和 WebView 内容可见。
11. 返回首次使用弹框。
12. 在弹框内容区域按元素内比例点击隐私政策链接。
13. 断言隐私政策页面标题和 WebView 内容可见。
14. 返回首次使用弹框。
15. 点击同意并继续。
16. 断言进入登录页。

验证点：

- 首次启动会展示个人信息保护条款弹框。
- 取消后重新打开仍会展示弹框。
- 用户协议和隐私政策链接可打开并返回。
- 同意并继续后进入登录页。

注意事项：

- 该用例执行 `clearAppData` 后会清理协议、登录和本地业务状态，因此不放入常规全量计划；如未来确需纳入聚合计划，必须作为最后一个独立 stage。
- 协议链接点击目前依赖 `common.authDialogContent` 的元素内相对比例；修改弹框文案或布局后需要重新校准。
